## 4. 各 Phase 详细计划

### Phase 5 — 数据源完善（v0.5.0，预估 8~12 人日）

#### T5.1 DuckDB 接入 UnifiedDataProvider 缓存层（1 人日）

**目标**: 将 store/duckdb.py 的 store/load 方法与 api.py 的 UnifiedDataProvider 打通

**任务**:
- [ ] 在 UnifiedDataProvider 中集成 DuckDB 作为缓存后端
- [ ] 数据获取流程改为: MemoryCache → DuckDB → HTTP 源
- [ ] 实现缓存过期策略（L2 缓存写回）
- [ ] 测试覆盖缓存命中和未命中路径

**验收**: `dc.get("RB", DataType.OHLCV)` 第二次调用命中 DuckDB 缓存

#### T5.2 国家统计局/央行宏观数据源（1.5 人日）

**目标**: 新增 2 个官方宏观数据源，与东方财富构成降级链

**任务**:
- [ ] 创建 `macro/providers/national_bureau.py` — 国家统计局 CPI/PPI/GDP
  - 使用 httpx 抓取 stats.gov.cn 公开数据
  - 解析 JSON/HTML 响应
  - 支持 indicator 参数筛选
- [ ] 创建 `macro/providers/pboc.py` — 央行 LPR/M2
  - 使用 httpx 抓取 pbc.gov.cn 公开数据
  - 解析时间序列
- [ ] 更新 `macro_provider.py` 降级链: 统计局 → 央行 → 东方财富 → Cached
- [ ] 编写 mock 测试覆盖两个新 provider

**验收**: `dc.get("*", DataType.MACRO, {"indicator": "cpi"})` 返回统计局数据

#### T5.3 期货基本面数据抓取（3 人日）

**目标**: 实现对交易所官网和生意社的 HTTP 抓取

**任务**:

- [ ] 更新 `futures_provider.py` 降级链
- [ ] 编写 mock 测试覆盖新 provider

**验收**: `dc.get("RB", DataType.FUTURES_BASIS)` 返回真实基差数据

#### T5.4 国信证券数据源（1.5 人日）

**目标**: 创建 guosen provider 骨架，完成 API 集成

**任务**:
- [ ] 创建 `equity/providers/guosen.py`
